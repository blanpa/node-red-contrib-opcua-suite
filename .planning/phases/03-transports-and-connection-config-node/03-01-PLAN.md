From lib/transports/base-transport.js (this plan creates it):
```javascript
/**
 * Abstract base class for all PubSub transport adapters.
 * Concrete subclasses MUST override connect, close, and send.
 *
 * Events emitted (per D-03):
 *   - "connected"      — transport ready to send/receive
 *   - "disconnected"   — transport closed (clean or unexpected)
 *   - "reconnecting"   — connection-oriented transports only (MQTT); UDP never emits
 *   - "error"          — Error or createError() shape; non-fatal, non-blocking
 *   - "message"        — (Buffer payload, optional metadata) — receive path (D-04)
 */
class BaseTransport extends EventEmitter {
  constructor(config) { super(); this.config = config; }
  async connect() { throw new Error("BaseTransport.connect() not implemented"); }
  async close()   { throw new Error("BaseTransport.close() not implemented"); }
  send(payload, opts) { throw new Error("BaseTransport.send() not implemented"); }
}
module.exports = { BaseTransport };
```

API contract (locked, do NOT change in 03-02/03-03/03-04):
- `connect()` returns Promise; resolves once `connected` event has been emitted.
- `close()` returns Promise; resolves once underlying socket/client is fully torn down. MUST be idempotent.
- `send(payload, opts?)` accepts `Buffer | Buffer[]` per D-02. Caller does NOT loop — transport does
  `Array.isArray(payload)` dispatch internally. `opts` is reserved (currently undocumented;
  03-03 uses it for per-publish QoS override, 03-02 ignores it).
- Subscriber path uses `transport.on('message', (buffer, metadata?) => ...)` per D-04. No
  `receive(callback)` setter, no async iterator.
