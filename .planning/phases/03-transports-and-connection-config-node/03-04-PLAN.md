<objective>
Implement the `opcua-pubsub-connection` Node-RED config node + editor HTML per CFG-01 and CFG-02.
This node owns the transport lifecycle: ref-count + 500ms grace timer + status fan-out to worker
nodes (Phase 4 Publisher/Subscriber). It mirrors the proven `nodes/opcua-endpoint.js` pattern with
the grace-timer addition per D-05/D-06/D-08.

Purpose: The config node is the integration seam between user-facing editor (transportType, broker
URL, multicast group, PublisherId, credentials) and the lib/transports/ classes. Phase 4
Publisher/Subscriber will resolve a config-node reference via `RED.nodes.getNode(config.connection)`
and call `acquireTransport()` / `releaseTransport()` / `registerStatusCallback()`.

Output: `nodes/opcua-pubsub-connection.js` (config-node module), `nodes/opcua-pubsub-connection.html`
(editor UI), `test/opcua-pubsub-connection.test.js` (Mocha suite using the createRED() mock pattern
from `test/connection-sharing.test.js`).

    `node.on("close", async function(removed, done) { ... })`:

    **WARNING — DO NOT FOLLOW RESEARCH.md PATTERN 7 VERBATIM FOR THIS HANDLER.**
    Pattern 7 in 03-RESEARCH.md shows a one-argument signature `async function(done)`. That is
    WRONG for Node-RED config nodes — the runtime invokes the handler with `(removed, done)`
    where `removed` is a boolean and `done` is the callback. The one-argument form receives
    `removed` as `done`, never calls the real callback function, and Node-RED hangs on close —
    silently breaking ROADMAP success criterion #5. Use the two-argument form here verbatim;
    test #16 already validates `(false, doneCb)`.

    - Cancel grace timer if pending.
    - If `_sharedTransport`: `await _sharedTransport.close()` in try/catch.
    - Null out `_sharedTransport`, reset `_refCount = 0`, clear `_statusCallbacks`.
    - Call `done()` — synchronous from Node-RED's perspective (close completes before next deploy).
    - **Acceptance criterion**: this MUST call `socket.close(done)` (UDP) or `client.end(false, {}, done)` (MQTT) under the hood — already structurally guaranteed by Plans 03-02/03-03's close() impls. ROADMAP success criterion #5.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Node-RED editor → config node | User-supplied values (brokerUrl, multicastGroup, publisherId, credentials) flow into transport constructors |
| Config node → log subsystem | Diagnostic logs may inadvertently include credentials (password) — must be redacted at the source |
| Worker node → config node API | Phase 4 Publisher/Subscriber call acquireTransport / releaseTransport / registerStatusCallback — misuse must not crash siblings |
| publisherId field → MQTT topic concat | publisherId is user input; concatenated into MQTT topic by MqttTransport. Must be validated BOTH at transport layer (Plan 03-03) AND surfaced in editor info text so user sees the constraint |
| Cert routes → file system | registerCertRoutes exposes upload endpoints; cert-store.js (Phase 1) handles sanitization — this plan only invokes it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03 | Information disclosure (CWE-532) | `node.error` / `node.warn` / `node.log` | mitigate | `_redactConfig(cfg)` helper returns shallow-copy with `password` (and `userName` for paranoia) removed before any log call. Connect-failure errors are passed through fan-out as Error objects; the underlying mqtt client's error.message MAY contain partial connect-string info but we never log credentials directly. Verified by Task 1 test #17 (sets credentials.password = "SECRET", triggers an error log path, asserts "SECRET" does NOT appear in node.error args). |
| T-03-04 | Tampering / Injection (CWE-74) | `publisherId` → MQTT topic concat | mitigate | Two-layer defense: (1) Plan 03-03's `_buildTopic` rejects `/`, `+`, `#`, control chars at transport layer (THIS layer feeds publisherId there). (2) Editor info text (this plan's HTML help panel) explicitly states the forbidden character set so users see the constraint before save. UDP path does NOT concatenate publisherId into a routable string, so the threat applies only on the MQTT side; covered structurally. |
| T-03-07 | Resource exhaustion / Socket leak (CWE-404) | `node.on('close', ...)` | mitigate | `node.on('close', async function(removed, done) { if (graceTimer) clearTimeout; if (sharedTransport) await sharedTransport.close(); done(); })`. Ensures: (a) pending grace timer is canceled (no zombie close at 500ms after Node-RED removed the node), (b) underlying socket.close(done) / client.end(false,{},done) completes BEFORE done() is called (synchronous Node-RED close — ROADMAP success criterion #5). Verified by Task 1 test #16 (simulates close event, spies on transport.close, asserts done called). |
