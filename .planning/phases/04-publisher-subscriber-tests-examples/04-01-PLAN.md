## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Node-RED editor → publisher node | User-supplied WriterGroup / DataSetWriter / PublishedDataSet config (intervals, ids, field types, writers JSON) flows into the Phase 2 factories and then onto the wire |
| Inbound `msg.payload` → DataSetMessage fields | Untrusted flow data keyed by field name is mapped into Variant values and encoded; missing fields must be omitted, extra/wrong-typed values must not crash the node |
| Publisher → shared transport (config node) | acquireTransport / releaseTransport / register/unregisterStatusCallback must be paired correctly so the ref-counted shared transport is not leaked or double-released |
| Cyclic setInterval → process lifetime | A leaked interval across redeploy keeps publishing after the node is gone (Pitfall: leaked timers) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Denial of Service (CWE-404, resource/timer leak) | cyclic `setInterval` | mitigate | The interval is stored on `node._interval` and `node.on("close", removed, done)` calls `clearInterval(node._interval)` before `done()`. Verified by Task 2 test 17 (tick after close yields no further sends). This is the publisher-side companion to Phase 3's transport-level leak guards. |
| T-04-02 | Denial of Service / crash (CWE-248, unhandled exception) | `node.on("input")` and the cyclic tick | mitigate | Both the input handler and the interval callback wrap frame-build + encode + send in try/catch; failures become `node.error(message)` + red status, never an uncaught throw that would kill the flow or leak the transport ref. Verified structurally + by Task 1 tests on the config-error path. |
| T-04-03 | Tampering / spec-violation (CWE-20, malformed wire frame) | factory build at construct + UDP-JSON combination | mitigate | (a) WriterGroup/DataSetWriter/PublishedDataSet are built through the Phase 2 validating factories — invalid ids/intervals/fields throw and are surfaced as node.error + red status with NO transport acquired (Task 1 tests 4). (b) D4-03 UDP-JSON is hard-rejected at startup (Task 1 test 5) so an unshippable transport/encoding combo never reaches the wire. |
| T-04-04 | Information / integrity (CWE-665, fabricated field data) | DataSetMessage field map | mitigate | Missing fields in `msg.payload` are OMITTED from the frame, never defaulted/fabricated (D4-05). Verified by Task 1 test 9. Prevents the subscriber from receiving phantom values for fields the publisher did not actually have. |
