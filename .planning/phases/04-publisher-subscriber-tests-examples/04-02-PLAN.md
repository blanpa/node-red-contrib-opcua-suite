<objective>
Implement the `opcua-subscriber` Node-RED worker node per SUB-01, SUB-02, STAT-01. The node
references an `opcua-pubsub-connection` config node, declares ONE `DataSetReader` filter
(publisherId / writerGroupId / dataSetWriterId, at least one required), registers its own
`message` listener on the shared transport, decodes each NetworkMessage with the encoding-selected
decoder, filters + ConfigurationVersion-checks each DataSetMessage, and emits one `msg` per matched
message in the exact D4-09 shape.

Purpose: This is the receive half of the PubSub loop. Plans 04-03 (round-trip + redeploy tests)
and 04-04 (example flows + README) depend on the config property names and the emitted `msg` shape
documented in the `<interfaces>` block below. The node MUST mirror the worker-node skeleton of
`nodes/opcua-event.js` (connection resolve, status callback, register/unregister, close cleanup)
and honor the locked decisions D4-01, D4-02, D4-03, D4-07, D4-08, D4-09, D4-10.

Output: `nodes/opcua-subscriber.js` (worker node), `nodes/opcua-subscriber.html` (editor UI),
`test/opcua-subscriber.test.js` (Mocha suite using the hand-rolled `createRED()` mock from
`test/opcua-nodes.test.js` plus a fake EventEmitter transport), and the `opcua-subscriber`
registration line in `package.json`.

<!-- Emitted msg shape, per matched DataSetMessage (D4-09 — EXACT): -->
```javascript
{
  payload:         { [fieldName]: <decodedScalar> },  // Variant/DataValue wrapper removed
  publisherId:     <from NetworkMessage.publisherId>,
  writerGroupId:   <from NetworkMessage.groupHeader.writerGroupId>,
  dataSetWriterId: <from DataSetMessage.dataSetWriterId / payloadHeader>,
  sequenceNumber:  <NetworkMessage.groupHeader.sequenceNumber ?? DataSetMessage.sequenceNumber>,

  statusCode:      <DataSetMessage.status ?? 0>,        // 0 = Good
  encoding:        "uadp" | "json",
  transport:       "udp" | "mqtt",
  topic:           <MQTT only: metadata.topic; key OMITTED entirely for UDP>
}
```

<!-- Decoder API consumed (lib/uadp-encoder.js, lib/json-encoder.js) — see 04-RESEARCH.md §uadp/§json: -->
```javascript
// UADP: decodeNetworkMessage(buffer, opts?) -> NetworkMessage; throws createError on malformed.
// JSON: decodeNetworkMessage(jsonString, opts?) -> NetworkMessage; takes a STRING (Buffer.toString() first).
//
// Decoded NetworkMessage model (both decoders, per 04-RESEARCH.md):
//   { publisherId?, groupHeader?: { writerGroupId, sequenceNumber, ... },
//     payloadHeader?: { dataSetWriterIds: number[] }, timestamp?: Date, payload: DataSetMessage[] }
// Decoded DataSetMessage model:
//   { dataSetWriterId?, messageType, sequenceNumber?, status?,
//     configurationVersion?: { major, minor }, timestamp?: Date,
//     fields: { [name]: { dataType, value } | { value: { dataType, value }, statusCode?, ... } } }
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Network → subscriber decoder | Untrusted UADP/JSON bytes arrive on the shared transport's `message` event and are fed to decodeNetworkMessage. Malformed/hostile bytes must not crash the node or the shared transport. |
| Decoded message → Node-RED flow | Decoded field values become `msg.payload` consumed by downstream nodes. Attacker-controlled field names/values flow into the flow graph. |
| publisherId/writerGroupId/dataSetWriterId filter | Editor-supplied filter governs which messages are emitted; a wrong/empty filter must fail closed (FILTER_REQUIRED) rather than emit everything. |
| Shared transport listener registry | The subscriber adds a listener to a ref-counted transport shared with sibling subscribers; failure to remove it on close leaks listeners and replays messages to dead nodes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-S1 | Denial of service (CWE-248, unhandled throw) | `onMessage` listener / decodeNetworkMessage | mitigate | The entire handler body is wrapped in try/catch; a decode error (malformed bytes) is surfaced as `node.error` and the listener returns normally — it NEVER throws out of the EventEmitter callback, so the shared transport and sibling subscribers stay alive (D4-07). Verified by Task 1 test 10 (garbage buffer does not throw; subsequent valid message still emits). |
| T-04-S2 | Resource exhaustion / listener leak (CWE-404) | `node.on("close")` cleanup ordering | mitigate | close handler calls `transport.removeListener("message", onMessage)` BEFORE `releaseTransport()`, and `unregisterStatusCallback`. Guarantees the shared ref-counted transport sheds this node's listener on every redeploy even when a sibling keeps the transport alive. Verified by Task 1 test 11 (listenerCount 0 after close, ordering asserted) and test 13 (20-cycle redeploy leaves 0 listeners). |
| T-04-S3 | Spoofing / message injection (CWE-290) | DataSetReader filter | mitigate | The reader requires at least one of publisherId/writerGroupId/dataSetWriterId (Phase 2 FILTER_REQUIRED); a reader with no filter fails closed at construct time (node.error + red status, no transport acquired) rather than emitting every received NetworkMessage. Filter compare drops non-matching publisher/group/writer ids silently. Verified by Task 1 test 3 (no-filter rejected) and test 8 (non-match skipped). |
| T-04-S4 | Tampering / silent data corruption (CWE-345) | ConfigurationVersion check | mitigate | A matched DataSetMessage whose ConfigurationVersion differs from the reader's expected version is dropped AND raises a VISIBLE `node.error` — it is never silently emitted as if schema-valid (D4-08). This is deliberately distinct from the silent filter-skip path. Verified by Task 1 test 9. |

Note: no `security_enforcement` key in workflow config; threat model included as defensive practice
because the subscriber is a new network-input trust boundary (untrusted PubSub bytes → Node-RED flow).
Transport-level datagram/topic threats (UDP reassembly DoS, MQTT TLS/topic injection) are owned by
Phase 3 plans 03-02/03-03 and are not re-litigated here.
</threat_model>
